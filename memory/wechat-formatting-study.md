# WeChat 公众号排版与配图研究笔记

## 一、已掌握的排版标准

### 配色体系（当前使用）
- **主色调**：青色 `#0f766e`（标题边框、强调）
- **深青**：`#134e4a`（三级标题）
- **背景**：白色 `#fefefe`（防过滤）
- **深色背景**：`#0f172a`（代码块）
- **正文**：`#374151`，16px，行高 1.8
- **辅助文字**：`#6b7280`，14px

### HTML 约束（必须遵守）
- 所有样式内联，不支持外部 CSS
- 列表展平为前缀段落（`• ` 或 `1. `），避免 `ul/ol/li`
- 图片添加 `width:100%`
- 链接使用 `content_source_url` 作为稳定点击路径
- 不使用 table、flex、grid、script

### 布局规则
- 标题底部加粗 + 底部边框（2px solid #0f766e）
- 二级标题：左边框 4px solid #0f766e
- 段落间距：margin 12px 0，行高 1.8
- 引用块：左边框 + 浅灰背景
- 代码块：深色背景 + 浅色文字

## 二、内容结构最佳实践

### 开篇引导块（必备）
- 一句话说明今天的主题和价值
- 简短的"适合谁看"说明
- 视觉上用引用块或居中样式

### 文章主体节奏
```
[引导块] → [章节标题] → [内容段落] → [配图] → [核心观点高亮] → [分隔线]
```

### 每篇文章的黄金结构
```
### 标题（带 emoji）

[配图]

**热度**：X 分

**发生了什么**：[2-3 句核心事件]

**为什么值得看**：[1-2 句深度分析]

---
```

### 结尾行动建议（增强互动）
```
## 💡 行动建议

- 具体可行的建议 1
- 具体可行的建议 2
- 如果感兴趣...（引导阅读原文）
```

## 三、配图 Prompt 模式库

### 通用科技资讯插画
```
现代科技媒体风格插画，适合微信公众号正文配图。
主题：{title}。
画面重点：{key_point}。
风格：{style}（见下方分类）、蓝青色主调、简洁大气。
不要文字，不要水印，不要 logo。
```

### 风格分类

| 内容类型 | 推荐风格 | Prompt 关键词 |
|---------|---------|-------------|
| AI/机器学习 | 科技感、数字抽象 | "数字神经网络、数据流、AI 芯片、科技蓝" |
| 开发工具 | 终端界面、代码感 | "终端界面、代码编辑器、开发者桌面" |
| 硬件/物理 | 现代工业、精密感 | "精密仪器、现代工厂、硬件设备" |
| 数据分析 | 信息图、大屏感 | "数据可视化、大屏仪表盘、信息图表" |
| 生活/创意 | 温暖、有趣 | "温暖配色、手绘质感、生活场景" |

### 配图尺寸
- 正文配图：1536 × 1024（3:2，横版）
- 封面图：1536 × 1024（可裁剪为 16:9 或 2.35:1）

## 四、各类内容的 Prompt 模板

### 1. 事件类（HN Top 5 类型）
```
现代科技媒体风格，{主题}相关插画。
画面元素：{具体事件描述}，{关键人物/产品}，{背景环境}。
氛围：新闻感、技术感、权威感。
配色：蓝青色系。
```

### 2. 工具/框架类（GitHub Trending 类型）
```
{工具名称} 的核心概念插画。
画面展示：{工具解决的问题}、{使用场景}、{界面截图或示意图}。
风格：科技媒体风格、蓝青色调、现代感。
不要文字。
```

### 3. 教程/指南类
```
教程配图，{主题}。
画面：步骤示意图、{工具界面}、{最终效果预览}。
风格：清晰易懂、现代简洁、步骤感。
```

### 4. 观点/分析类
```
深度分析文章配图，{主题}。
画面：{核心观点可视化}，{对比图}，{数据展示}。
风格：严肃、专业、图表感。
```

## 五、结构优化 Checklist

### 发文前必查
- [ ] 开篇有引导句（价值说明）
- [ ] 每个条目有配图
- [ ] 标题简短有力（不超过 20 字）
- [ ] 每条有"发生了什么"和"为什么值得看"
- [ ] 结尾有行动建议或延伸阅读
- [ ] 分隔线使用 `---`
- [ ] 列表全部展平（不用 ul/ol）
- [ ] 代码块使用 `<pre><code>`
- [ ] 配图宽高比一致

### 避免的陷阱
- ❌ 长段落（超过 5 行没有配图或小标题）
- ❌ 连续 bullet 没有层次
- ❌ 所有条目结构完全一样（轻微变化增加可读性）
- ❌ 标题太长（信息密度过高）
- ❌ 没有行动建议（读者不知道看完要做什么）

## 六、GitHub Trending 文章专属优化

### 标题公式
```
{项目简称}：{一句话描述解决的问题}
```
例：`MoneyPrinterV2：自动化"油管视频+推文+联盟营销"引流变现`

### 内容块结构
```
### X. {项目名}

![配图](./generated-body-auto/日期/story{N}.png)

**今日 ★**：{stars_today} | **总 ★**：{total_stars} | **语言**：{language}

**发生了什么**：{AI 生成的深度解读，包含项目背景、解决的问题、技术亮点}

**为什么值得看**：{AI 生成的分析，说明对开发者的价值}

**链接**：{GitHub URL}
```

### 配图 Prompt 优化
```
GitHub Trending 项目配图，主题：{项目名}。
核心功能：{一句话描述}。
技术亮点：{关键特点}。
风格：科技媒体插画、蓝青色调、简洁现代感。
不要文字不要水印。
```

## 七、后续改进方向

1. **建立分类 Prompt 库**：为 AI/开发工具/硬件/生活等不同类容制定专属 Prompt 模板
2. **配图风格一致性**：同一系列文章使用相似但不完全相同的风格
3. **增加数据可视化**：数字类内容尝试生成信息图风格的配图
4. **优化开篇引导**：每篇文章设计独特的开篇 hook
5. **结尾互动设计**：增加"你怎么看"类的互动引导
