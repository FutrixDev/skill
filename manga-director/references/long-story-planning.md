# Long Story Planning

Use this reference when the source is a novel, novella, multi-chapter story, long script, or any story with major time jumps.

## Whole Story First

Before chapter storyboards, create `storyboard/series-plan.json`.

The plan must capture:

- whole-story premise
- central conflict
- core themes
- emotional spine
- ending state
- recurring locations and objects
- political, magic, or world rules
- unresolved threads carried between chapters

Do not plan chapter 1 as if later chapters do not exist. Early chapters must preserve setups, objects, wounds, promises, relationships, and world rules needed later.

## Chapter Split

Each chapter plan should define:

- source scope
- story function
- main beats
- details that must not be lost
- continuity coming in
- continuity going out
- estimated page count
- character stages used in the chapter

If a chapter has too many A-level details for its page count, propose a longer page count or split the chapter.

## Character Stages

Large stories often need multiple reference sheets for the same person.

Create a new character stage when any of these change:

- age
- social role or rank
- outfit or armor
- injury, scar, illness, exhaustion, or dirt
- emotional baseline
- physical build
- hair state
- major prop set
- living situation or status

Examples:

- `shenqiao_child_13`
- `shenqiao_youth_20`
- `shenqiao_official_30`
- `luqingheng_refugee_16`
- `luqingheng_adult_commoner`

Every page and panel that uses a staged character must say which stage it references. Do not rely on the name alone.

## Chapter-to-Chapter Continuity

Track chapter outputs:

- unresolved questions
- injuries and costume changes
- recurring props
- relationship changes
- location damage or seasonal changes
- new facts learned by the reader
- future callbacks created by this chapter

Use those outputs as the next chapter's continuity input.
