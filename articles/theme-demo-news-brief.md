# 用 news-brief 做一篇 GitHub Trending 快讯

::: lead
这个主题的目标不是优雅，而是快。读者应该在一屏之内知道今天最值得看的项目是什么，以及为什么要看。
:::

## 今日观察

今天如果是做 Trending 或 HN Top 5，首屏应该先给重点，不要先上背景故事。

::: summary
资讯型内容最怕拖。这个主题应该把“重点、变化、行动建议”三件事放到最前面。
:::

## Top 3 值得看

1. 第一个项目：概念新，扩散快。
2. 第二个项目：落地强，开发者容易上手。
3. 第三个项目：讨论大，但含金量要再观察。

::: tip
如果是日报，段落最好比教程更短，避免一屏内出现太多解释性文字。
:::

## 编辑角度的风险

::: warning
资讯类文章很容易被写成“链接堆”。每一条都要有一句判断，不然就只是搬运。
:::

## 对比速览

::: comparison
| 写法 | 读者感受 | 风险 |
| --- | --- | --- |
| 纯链接列表 | 很快扫完 | 缺乏判断 |
| 带 1 句点评 | 信息更有价值 | 编辑工作量更高 |
:::
