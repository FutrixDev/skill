# 用 warm-magazine 写一篇有判断的技术随笔

::: lead
有些技术文章不是为了教人点哪里，而是为了帮人想清楚：为什么要做、为什么现在做、为什么不是另一条路。
:::

## 这种主题适合什么

适合写判断、写取舍、写过程里的犹豫，也适合把技术问题讲得更像人的问题，而不是纯配置问题。

::: summary
如果文章里有明显的个人语气、经验判断、价值排序，这个主题会比冷冰冰的科技风更合适。
:::

## 我更喜欢的写法

不是把所有命令都贴出来，而是把真正关键的决策点说清楚：为什么选这套链路，为什么不换别的 provider，为什么发布链路要先稳后花。

::: note
公众号读者并不总是在找“源码”，很多时候他们是在找一个足够可信的判断。
:::

::: warning
如果一篇观点文堆了太多 bullet、太多代码块、太多表格，它就会开始失去呼吸感。
:::

## 放在一起看

::: comparison
| 写法 | 气质 | 更适合 |
| --- | --- | --- |
| 冷静技术文 | 准、快、硬 | 教程、评测 |
| 温暖杂志风 | 慢一点，但更有人的判断 | 观点、随笔、长文 |
:::
